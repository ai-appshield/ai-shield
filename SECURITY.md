# Security Policy

## Supported Versions

We release patches for security vulnerabilities for the following versions:

| Version | Supported          |
|---------|--------------------|
| 1.5.x   | ✅ Yes             |
| < 1.5   | ❌ No              |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, report them privately via GitHub's built-in security advisory system:

1. Go to the [Security tab](https://github.com/ai-appshield/ai-shield/security)
2. Click **"Report a vulnerability"**
3. Fill in the details — what you found, how to reproduce it, and the potential impact

You can also email: **security@traghmtech.com**

## What to Expect

- **Acknowledgment** within 48 hours
- **Assessment** within 7 days
- **Resolution timeline** communicated as soon as impact is assessed
- **Credit** given in the release notes if you'd like it

## Scope

AI AppShield is a documentation framework — the primary security concern is supply chain integrity (i.e., ensuring template files and directive snippets have not been tampered with). We take any issue that could cause an AI agent to behave unsafely or insecurely seriously.
