# Contributing to AI AppShield

Thank you for helping make AI-assisted development more reliable for everyone.

## What We Need Most

- **Tool-specific directive snippets** — If you use an AI coding tool not listed in `/DIRECTIVE`,
  create a snippet file following the existing format.
- **Framework starters** — Language/framework-specific `conventions.md` templates
  (Python/Django, Go, Swift/SwiftUI, Ruby on Rails, etc.)
- **Production examples** — Anonymized real-world `/docs` structures that show how AppShield
  scales to complex projects.
- **Agent behavior findings** — If you discover a reliable pattern for getting AI agents to
  follow instructions more consistently, document it.
- **Translations** — Core documents in languages other than English.

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b feature/your-contribution`
3. Make your changes
4. Ensure your addition follows the existing format and tone
5. Submit a pull request with a clear description of what you added and why

## Guidelines

- Keep template files practical — every line should serve a real use case
- No vendor lock-in — templates should work across AI tools, not just one
- Real beats theoretical — contributions from production use are prioritized
- Credit is given — significant contributions are acknowledged in release notes
